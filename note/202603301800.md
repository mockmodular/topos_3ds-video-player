# 2026-03-30 下屏新增 CPU 实测频率 + 四核占用率显示

## 背景
用户要求在下屏诊断区显示当前 CPU 运行频率和四核占用率，且数据来源要直接读硬件。

## 实现

### CPU 实测频率（y=118，绿色）
- 数据来源：`OS_SharedConfig->timeref[timeref_cnt & 1].sysclock_hz`（虚拟地址 `0x1FF81000`）
- `sysclock_hz` 是 PTM 服务将 ARM11 tick 计数器与 RTC 实时对比后写入的实测值，是第一手硬件数据
- 除以 1,000,000 得 MHz，N3DS→~804，O3DS→~268，动态反映实际运行模式
- 显示格式：`CPU: 804MHz`

### 四核占用率（y=127，黄色）
- 数据来源：`Util_cpu_usage_get_cpu_usage(0~3)`（已有模块，已 init）
- NaN（核不存在或未 init）处理为 0
- 显示格式：`C0:XX% C1:XX% C2:XX% C3:XX%`

## 探索过程与结论
- `osGetSystemClockSpeed()` 不存在于此版本 libctru
- `osSetSpeedupEnable` 只有 set 无 get
- `app.rsf` 写死 `CpuSpeed: 804MHz`，但运行时可通过 `osSetSpeedupEnable(false)` 降回 268MHz，模型推算无法反映动态变化
- `OS_SharedConfig->timeref[x].sysclock_hz` 是唯一能从 usermode 读到的实测时钟频率
- ARM11 MPCore 所有核共享同一时钟域，无法获取单独每核频率；曾实现逐核显示但四值相同，已删除

## L2 缓存显示
- 最终决定不显示：usermode 无法直接读 PL310 寄存器（物理地址 0x17E10000 未映射到进程空间），只能间接推断，用户选择放弃
