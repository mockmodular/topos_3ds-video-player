# O3DS 视频解码移到 core 1（2026-03-30）

## 背景

O3DS 播放 MPEG2 SBS 时 core 0 达 100%，core 1 仅 25%，偶发微卡。
原因：core 0 同时跑 主线程渲染 + 音频软解 (HIGH) + 视频解码 (HIGH) + FFmpeg slice 0 (HIGH)，I 帧时爆 core 0 → 渲染被抢 → 微卡。

## 改动

### video_player.c ~2780
O3DS 分支 `Vid_decode_video_thread` 从 core 0 改为 core 1：
```c
// 前：core 0
vid_player.decode_video_thread = threadCreate(Vid_decode_video_thread, ..., DEF_THREAD_PRIORITY_HIGH, 0, false);
// 后：core 1
vid_player.decode_video_thread = threadCreate(Vid_decode_video_thread, ..., DEF_THREAD_PRIORITY_HIGH, 1, false);
```

### vid_settings.c ~77
O3DS fake_pthread 可用核列表只开 core 1（原来 core 0 + core 1）：
```c
// 前：
for(uint8_t i = 0; i < 2; i++) { frame_cores[i] = ...; slice_cores[i] = ...; }
// 后：
frame_cores[1] = Util_is_core_available(1);
slice_cores[1] = Util_is_core_available(1);
```
效果：FFmpeg MPEG2 slice 线程全部绑 core 1，round-robin 只有1个核 → 实质单线程，消除 core 间 slice 同步开销。

## 结果

| Core | 改前 | 改后 |
|---|---|---|
| 0 | 渲染+音频+视频解码+slice 0 | 渲染+音频 |
| 1 | convert+read_packet+slice 1 (25%) | 视频解码+convert+read_packet+所有slice (扛全部视频) |

编译 **0 error 0 warning**。
