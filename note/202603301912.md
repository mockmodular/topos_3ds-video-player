# 2026-03-30 FFmpeg ARMv6/ARMv5TE ASM 重编

## 发现
原版 FFmpeg for 3DS 构建脚本（`ffmpeg_build.md` / `build_3ds.sh`）带有 `--disable-asm`，导致：
- `HAVE_ARMV6 0` / `HAVE_ARMV5TE 0` / `HAVE_VFP 0`
- `libavcodec.a` 中只有纯 C 目标文件，以下 ARMv6/ARMv5TE 汇编全部缺失：
  - `simple_idct_armv6.o` — MPEG-2 IDCT（最大瓶颈）
  - `hpeldsp_armv6.o` / `hpeldsp_init_armv6.o` — 半像素插值（运动补偿）
  - `mpegvideo_armv5te.o` / `mpegvideo_armv5te_s.o` — MPEG-2 运动补偿
  - `idctdsp_armv6.o` / `idctdsp_init_armv6.o`
  - `pixblockdsp_armv6.o`、`me_cmp_armv6.o`、`videodsp_armv5te.o` 等

## 操作
1. 确认本机环境可用：`C:\devkitPro\msys2\usr\bin\bash.exe` + devkitARM r65 GCC 14.2.0
2. 用 `pacman -S --noconfirm gcc` 补装 msys2 host gcc（GCC 15.2.0），解决 configure "Host compiler lacks C11 support" 错误
3. 新建 `D:\3dsvideoplayermock\Topos0.01\build_ffmpeg_asm.sh`：路径指向 Topos0.01 目录的库，**去掉 `--disable-asm`**，PREFIX 指向新目录
4. FFmpeg 源码：`D:\3dsvideoplayermock\Topos0.01\ffmpeg`（作者 3ds 分支）
5. configure + make -j（约 5 分钟）

## 结果
- 新库：`D:\3dsvideoplayermock\Topos0.01\ffmpeg_asm_built\lib\`
- `HAVE_ARMV6 1` / `HAVE_ARMV5TE 1` / `HAVE_VFP 1`
- `libavcodec.a` 5.5MB → 6.1MB（+0.6MB 为 ARMv6/ARMv5TE .o）
- 已替换入 `topos.core0.01/library/lib/`
- 旧库备份：`D:\3dsvideoplayermock\Topos0.01\library_lib_backup_no_asm\`
- 项目 `make` 编译：**0 error 0 warning**

## 注意
- ARM11 (ARMv6K) 无 NEON，NEON-OBJS 不受影响（运行时不会调用）
- NEON 重编无意义，硬件根本没有该指令集
- 3DS 软解（MPEG-2 等）提升预计显著，需上机实测
