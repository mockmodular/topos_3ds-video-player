# Step 5c 完成 — 2026-03-30

## 变更内容

**目标**：`Vid_hid()` 的 CFM 执行段（约 466 行 HID 宏直驱 vid_player）替换为 enqueue→process 命令队列流程。

### 新增（video_player.c 静态函数）

| 函数 | 作用 |
|---|---|
| `vid_fill_layout(VidHidLayout *out)` | 从 `vid_player` 的所有 Draw_image_data 成员填 layout 快照 |
| `vid_fill_router_state(VidHidRouterState *out, bool is_new_3ds, double scroll_speed)` | 从 `vid_player` 状态填路由快照 |
| `vid_fill_locks(VidHidUiLocks *out)` | 从 `vid_player.seek_bar.selected` 等填 lock 快照 |
| `Vid_process_hid_cmd_queue(void)` | `while(Vid_cmd_pop)` + `switch(cmd.id)` 消费所有命令，执行对 vid_player 的写操作和 Util_queue_add 请求 |

### Vid_hid() 改动

- 删除：约 466 行 HID_*_CFM 宏直驱执行段（brightness/fullscreen/audio/controls/menu/seek 等）
- 保留：SEL 段（按钮高亮）、DESEL 段（解选）、Scroll 段（滚动惯性）、aptShouldJumpToHome 顶部段
- 新增：
  ```
  auto_full_screen_count 重置（同原逻辑，加守卫条件）
  vid_fill_layout / vid_fill_router_state / vid_fill_locks
  Vid_cmd_queue_reset → Vid_hid_enqueue → Vid_process_hid_cmd_queue
  move content（原有 Vid_hid_compute_move_deltas 块，保留）
  重填 router_state/locks → Vid_cmd_queue_reset → Vid_hid_enqueue_seek → Vid_process_hid_cmd_queue
  ```
