# 改动记录 2026-04-02

## 本次实例主要改动

---

### 1. Y2R_X2 SBS 直写纹理（零拷贝）

**文件**：`source/system/util/converter.c`、`include/system/util/converter.h`、`source/vid_worker.c`

**变更内容**：
- `Util_converter_y2r_yuv420p_sbs_to_rgba8888` 签名从 `(**left_rgba, **right_rgba, full_width, height)` 改为 `(*dst_left, *dst_right, tex_width, full_width, height)`
- 函数内部移除 free/linearAlloc，改为接受预分配目标指针
- `Y2RU_SetReceiving` 的 `transfer_unit = half_w*4*8`，`transfer_gap = (tex_width - half_w)*4*8`，硬件直写至纹理内存
- `vid_worker.c` Y2R_X2 SBS 转换调用侧：传 `sbs_l_tex->data`/`sbs_r_tex->data` 作为目标
- `vid_worker.c` Y2R_X2 SBS 上传块：删除 30次 memcpy_asm 循环，替换为 `C3D_TexFlush` ×2 + 直接 subtex 字段更新

**效果**：每帧节省 768KB CPU memcpy；每帧消除 2次 linearAlloc/free（原 48次/秒）；Y2R_X2 帧率提升至接近 NEONy2r 水平。上机验证通过，gap 参数输出侧生效正确。

### 2. 新增 2D Y2R_X2 直写函数

**文件**：`source/system/util/converter.c`、`include/system/util/converter.h`

**新函数**：`Util_converter_y2r_yuv420p_to_rgba8888_direct(yuv, dst, tex_width, width, height)`
- 单次 Y2R，`transfer_unit = width*4*8`，`transfer_gap = (tex_width - width)*4*8`
- 无 alloc/free，直写至调用方提供的纹理指针

**vid_worker.c 变更**：
- 2D Y2R_X2 转换调用：改用新函数，传 `images[0].c2d.tex->data`
- 2D Y2R_X2 上传块：改为 `C3D_TexFlush` + subtex 字段更新 + `Vid_large_texture_crop`（不再调 `Vid_large_texture_set_data`）

**路径矩阵（最终状态）**：

| 模式 | 3D（SBS） | 2D |
|---|---|---|
| Y2R_X2 | 2次直传，零 CPU 拷贝 ✅ | 1次直传，零 CPU 拷贝 ✅ |
| NEONy2r | 单 pass 全帧 + 30次 tile-row 分割 | 全帧 Y2R + Vid_large_texture_set_data 30次循环 |

### 3. O3DS 线程调度优化

**文件**：`source/vid_lifecycle.c`（O3DS `else` 分支，行 ~492-498）

| 线程 | 改动 |
|---|---|
| `decode_thread` | affinity C0 → **C1**（释放 C0 给视频解码独占） |
| `convert_thread` | 优先级 NORMAL → **HIGH**（防被读包/音频线程抢占） |

**动机**：旧版 decode_thread 在 C0 与 decode_video_thread 竞争；convert_thread NORMAL 被三个 HIGH 线程（读包/音频/解码路由）抢占，导致纹理上传延迟、偶发掉帧。

---

### 4. libmpeg2 可行性评估（结论：不建议现阶段实施）

- 3DS ARM11（ARM1176JZF-S）**无 NEON**，libmpeg2 的 NEON 优化无法利用
- 收益仅来自更低框架开销，估 5~10%
- 集成复杂度高（替换 FFmpeg MPEG2 解码后端），维护成本高
- 当前 C0 已被 `decode_video_thread` 独占，性价比不足以优先考虑

---

### 5. restart_playback_threshold 调整

从 48 → **50**（单位：帧数，即解码完 50 帧后退出缓冲状态开始播放）
