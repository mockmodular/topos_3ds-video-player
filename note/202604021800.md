# Seek 机制与进度条 UI 解耦（2026-04-02）

## 新增文件

| 文件 | 作用 |
|------|------|
| `include/vid_seekbar.h` | 进度条几何常量（X/Y/W/H）+ inline `VidSeekBar_hit_test()` + `VidSeekBar_touch_x_to_pos()` |
| `include/vid_seek_engine.h` | `VidSeekEngineView` 结构体 + 4个输入API + `get_view()` 输出API |
| `source/vid_seek_engine.c` | seek 引擎实现；includes 必须加 `system/util/err.h`（同 vid_worker.c 陷阱） |

## 修改文件摘要

- `include/vid_hid.h`：`VidHidLayout` 移除 `seek_bar`；`VidHidRouterState` 移除 `media_duration / media_current_pos / seek_pos / seek_ui_pos`
- `include/vid_hid_macros.h`：`VID_HID_SEEK_BAR_SEL` 和 `VID_HID_TOUCH_EMPTY_ENTER_FULL` 改用 `VidSeekBar_hit_test()`，断开对 `Draw_texture` 副作用几何的依赖
- `source/vid_hid.c`：drag 命令改为 `vid_push_i(VID_CMD_SEEK_BAR_DRAG, touch_x)`，移除 `L` 本地别名（宏新版不再用 `L.seek_bar`）
- `source/vid_input.c`：4个 seek 命令处理改调引擎API；`HID_SEEK_BAR_SEL` 改用 `VidSeekBar_hit_test()`；`vid_fill_layout/router_state` 简化
- `source/vid_draw.c`：seek 条绘制改用 `VidSeekEngine_get_view()` + `VID_SEEKBAR_*` 常量

## 架构变化

```
前：vid_hid.c 把 touch_x + media_duration 打包入命令
   vid_input.c 用魔法数字 5/310 计算位置 → 直接写 seek_pos_cache
   vid_draw.c 直接读 seek_pos_cache / seek_pos / media_current_pos + 硬编码 5/210/310/10

后：vid_hid.c 只传 touch_x（整数）
   vid_input.c 调 VidSeekEngine_on_bar_drag/commit/step_fwd/step_back
   vid_seek_engine.c 集中管理所有 seek 字段
   vid_draw.c 调 VidSeekEngine_get_view() + VID_SEEKBAR_* 常量
```

## 陷阱

- `VID_SEEKBAR_*` 常量必须用 `double` 字面量（`5.0` 而非 `5.0f`），否则 `-Wdouble-promotion`
- `vid_seek_engine.c` 必须显式 `#include "system/util/err.h"`（同 vid_worker 陷阱）
- `Draw_texture` 仍会写入 `vid_player.seek_bar` 的 x/y/x_size/y_size，但这些值现在不再被命中测试读取，属"只写不读"
