# 2026-03-29 21:00 — high_ram 机制分析 + Makefile 修改

## 发现：high_ram 版本机制

- **不是另一套代码**，同一个 `.elf`，只换 `.rsf` 打包配置
- `makerom` 打 CIA 时用 `app_high_ram.rsf` vs `app.rsf`
- 两个 RSF 的唯一差异：

| 参数 | normal (`app.rsf`) | high_ram (`app_high_ram.rsf`) |
|---|---|---|
| `SystemMode`（O3DS堆）| `64MB` | `80MB` |
| `SystemModeExt`（N3DS堆）| `124MB` | `178MB` |
| `CpuSpeed` | `804MHz` | `804MHz`（同） |
| `EnableL2Cache` | `true` | `true`（同） |
| `CanAccessCore2` | `true` | `true`（同） |

- N3DS 多 54MB，O3DS 多 16MB
- 无 `#ifdef HIGH_RAM` 条件编译，纯打包差异

## 知识点：N3DS 为何是 124MB 不是 128MB

- N3DS 物理多 128MB，但系统保留约 4MB 给 N3DS 专属系统模块（mvd、qtm 等）
- 128MB - 4MB = 124MB 可分配给应用
- `124MB / 178MB` 是固件写死的枚举档位，不可自定义

## Makefile 修改

- **改动**：`cia_all` 依赖从 `cia_normal_ram cia_high_ram` → `cia_normal_ram`
- `cia_high_ram` target 定义保留但不再触发
- 构建现在只输出 `xxx.cia`，不再输出 `xxx_high_ram.cia`
