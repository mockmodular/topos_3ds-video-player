# Y2RU stride split — SBS tile行切割消除（2026-03-30）

## 动机
SBS+Y2R路径每帧需要30次 tile 行 memcpy（768KB搬运）才能把整帧BLOCK_8×8数据拆成左右眼。
Y2RU API的`Y2RU_SetSendingY/U/V`有`gap`参数（每行读完后跳过的字节数），可以让硬件直接跳着读单眼数据，彻底省掉CPU搬运。

## 改动列表

### `include/system/util/converter.h`
- 新增声明：`Util_converter_y2r_yuv420p_sbs_to_rgba8888`
- 新增 disabled stub（`#else`块）

### `source/system/util/converter.c`（行~852）
新增函数 `Util_converter_y2r_yuv420p_sbs_to_rgba8888`：
- 接收全宽YUV420P（full_width×height），输出两个BLOCK_8×8单眼RGBA8888
- Pass 1（左眼）：Y/U/V src各从列0开始，gap=half_w/half_uv_w，input_line_width=half_w
- Pass 2（右眼）：Y/U/V src各偏移half_w/half_uv_w字节，相同gap
- 串行两次Y2R（svcWaitSynchronization确保完成后再发第二次）
- 内部`free`+`linearAlloc`管理输出缓冲（与原函数风格一致）

### `source/video_player.c`
1. **sbs_right_buf 分配（行~3215）**：`malloc` → `linearAlloc`（Y2R receive buffer必须物理连续）
2. **sbs_right_buf 释放（行~2496, ~3574）**：`free` → `linearFree`
3. **HW_CONVERSION分支（行~4722）**：在`is_sbs_3d && sbs_right_buf`条件下走新函数，else走原单眼函数
4. **SBS+Y2R上传分支（行~4759）**：删除30次tile行切割loop，直接上传已就绪的video（左眼）和sbs_right_buf（右眼）

## gap参数计算（800×240为例）
```
Y平面（800×240=192000字节）：
  左眼 src=yuv,     width=400, gap=400
  右眼 src=yuv+400, width=400, gap=400

U平面（400×120=48000字节）：
  左眼 src=yuv+192000,     width=200, gap=200
  右眼 src=yuv+192000+200, width=200, gap=200

V平面（400×120=48000字节）：
  左眼 src=yuv+240000,     width=200, gap=200
  右眼 src=yuv+240000+200, width=200, gap=200

Y2RU_ConversionParams.input_line_width = 400（半宽）
Y2RU_SetReceiving unit = 400*4*4 = 6400（沿用原公式）
```
